# Contributing

Thanks for helping improve Ascend Chan.

## Before You Start

- Read the setup steps in `README.md`
- Keep secrets out of the repo and do not commit private credentials or service-role keys
- Do not commit production keys, passwords, or personal project URLs unless they are intentionally public

## Development Guidelines

- Keep the project static-first and lightweight
- Prefer small, focused pull requests
- Match the current coding style in nearby files
- If you change the database contract, update `supabase_schema.sql`
- If behavior changes, update the relevant docs

## Pull Requests

- Describe what changed and why
- Include screenshots for visible UI changes when helpful
- Call out any Supabase schema or policy changes clearly
- Mention any manual test steps reviewers should run

## Security

If you find a security issue, please do not open a public issue first. Follow the process in `SECURITY.md`.
